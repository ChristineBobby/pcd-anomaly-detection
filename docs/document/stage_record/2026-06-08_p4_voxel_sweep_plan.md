# P4 PASDF Voxel Sweep Plan

## Scope

This run is a small diagnostic sweep, not a 40-class tuning pass.

Input baseline:

- Results CSV: `experiments/E1_pasdf_baseline/full_40cls/evaluation_results.csv`
- Run log: `experiments/E1_pasdf_baseline/full_40cls/run.log`
- Sweep root: `experiments/P4_registration_sweep`
- Summary output: `docs/document/stage_record/2026-06-08_p4_voxel_sweep_summary.md`
- CSV output: `docs/document/stage_record/2026-06-08_p4_voxel_sweep_summary.csv`

## Selected Classes

Primary registration-warning classes:

- `cap3`: lowest object AUROC and 8 Open3D warning events.
- `cap4`: second-lowest object AUROC and 6 Open3D warning events.

Control classes with low object AUROC but no Open3D warning in the P3 log:

- `tap1`: object AUROC below 0.8, official PASDF voxel size differs from default.
- `helmet2`: object and pixel AUROC both low, but no Open3D warning in P3 log.

## Sweep Grid

- `cap3`: `0.02, 0.03, 0.04, 0.05`
- `cap4`: `0.02, 0.03, 0.04, 0.05`
- `tap1`: `0.03, 0.04`
- `helmet2`: `0.03`

This gives 11 PASDF runs. The grid intentionally includes default `0.03`, official class-specific values, and nearby coarse alternatives for the two warning-heavy cap classes.

## Interpretation Rules

- If object AUROC improves and warnings decrease, registration parameter sensitivity is likely a real contributor.
- If warnings decrease but AUROC does not improve, the Open3D warning is a stability signal but not the dominant error source.
- If AUROC changes without warnings, inspect scorer behavior, GT distribution, and defect morphology rather than only registration.
- If a class stays low across all voxel sizes, move it to geometry/SDF residual analysis.
